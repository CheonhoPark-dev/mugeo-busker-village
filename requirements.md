# 기능 상세 요구 정의서 (Functional Requirement Definition)

## 1. 프로젝트 개요 (Project Overview)

- **프로젝트명**: 무거 버스커 마을 웹사이트 구축
- **단체명**: 무거 버스커 마을
- **목적**: 무거동 주민들이 기타를 매개로 소통하고 즐기는 커뮤니티 공간을 온라인으로 확장하여, 단체 소개 및 정보 공유, 참여를 유도함.
- **핵심 가치**: "기타를 잘치는 것 보다 즐기는 것에 진심인 모임"

## 2. 사용자 정의 (User Roles)

- **일반 방문자 (Visitor)**: 단체 정보를 확인하고, 연혁을 보고, 연락처를 확인하는 모든 사용자.
- **관리자 (Admin)**: (추후 확장 고려) 게시물 관리, 일정 관리 등을 수행하는 운영진.

## 3. 사이트 구조 (Site Map)

1.  **홈 (Home)**: 메인 비주얼, 핵심 슬로건, 최신 소식 요약
2.  **단체 소개 (About)**: 단체 상세 소개, 비전, 연혁 (History)
3.  **활동 소식 (Activities/News)**: 공지사항, 행사 정보 (플래시몹 등)
4.  **문의 (Contact)**: 연락처 및 SNS 링크

## 4. 상세 기능 요구사항 (Detailed Functional Requirements)

### 4.1. 공통 사항 (Common)

- **헤더 (Header)**: 로고(단체명), 글로벌 네비게이션 바 (GNB).
- **푸터 (Footer)**: 카피라이트, 간단한 연락처 정보.
- **반응형 웹 (Responsive Web)**: 모바일, 태블릿, PC 등 다양한 디바이스 지원.

### 4.2. 홈 (Home)

- **메인 배너 (Hero Section)**
  - 배경: 활동적인 버스킹 이미지 또는 따뜻한 일러스트.
  - **슬로건 표시**: "기타를 잘치는 것 보다 즐기는 것에 진심인 모임" (강조).
- **단체 요약 소개**: "무거동 주민이라면 누구나..." 문구의 일부를 발췌하여 흥미 유발.

### 4.3. 단체 소개 (About)

- **상세 소개 텍스트**:
  - "기타를 잘 치는 것보다 즐기는 것을 추구하는 모임입니다."
  - "무거동 주민이라면 누구나 부담 없이 참여하고 자유롭고 편하게 즐길 수 있습니다."
  - "기술이나 실력보다 진심과 몰입을 추구합니다."
  - "기타를 좋아하는 마음만 있다면 언제나 당신을 환영합니다."
- **연혁 (History)**: 타임라인 형태로 시각화.
  - **2025.11.23(일)**: 무거천 플래시몹 (핵심 사건).

## 5. 비기능 요구사항 (Non-Functional Requirements)

- **디자인 (Design)**:
  - 따뜻하고 친근한 마을 공동체 느낌.
  - 가독성 높은 폰트 사용.
- **접근성 (Accessibility)**: 고령층 주민도 쉽게 읽을 수 있도록 충분한 대비와 글자 크기 고려.
- **SEO (검색 엔진 최적화)**: '무거동 기타', '무거 버스커' 등으로 검색 시 노출되도록 메타 태그 설정.
